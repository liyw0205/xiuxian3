# 背包组件说明

背包负责玩家身上占负重的普通物品库存，并提供恢复类物品的直接使用入口。

物品设定上，背包主要承接世界物资：纯经济、药路、民生、建设、古物、战利品。角色成长道具通常进入纳戒、装备、武器库或源石体系，不按世界物资处理。

## 命令

```text
背包
使用 物品名
使用 物品名 数量
```

## 规则

- 背包物品占格子和负重，主要包括纯经济特产、战利品和部分占负重的世界物资掉落。
- 纯经济只指 11 个普通探险/跑商重合地点各 3 个特产，共 33 个；药路、民生、建设、古物、战利品按六大类单独归属，不混入跑商特产。
- `背包` 会展示当前负重、物品数量、单件重量和分类。
- `使用` 只允许直接消耗恢复类物品。
- 如果物品在背包且分类不是恢复类，会提示统一出售入口：指定物品用 `出售 物品名 数量`，清理背包用 `自动出售`。
- 如果物品在纳戒且分类是恢复类，也可以通过 `使用 物品名` 或 `使用 物品名 数量` 消耗。
- 不写数量时默认使用 1 个；例如 `使用 福袋 5` 会一次打开 5 个福袋。
- 宝石、技能书、洗髓液、开孔器不能走 `使用`，必须走各自玩法命令。
- 背包放入物品时会检查负重、堆叠上限和物品种类上限。

## 扩展

- 新增背包物品：改 `item_defs` 初始化数据。
- 需要可直接使用的物品，分类必须是 `恢复类`，并在 `item_effects.py` 中支持对应效果。
- 新增普通世界物资时，必须先确认它属于纯经济、药路、民生、建设、古物、战利品之一，并给 `使用` 提示配置对应流向。
- 负重、格子和堆叠检查在 `CoreService.can_add_backpack_conn()`。
