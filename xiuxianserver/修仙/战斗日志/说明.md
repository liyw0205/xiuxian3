# 战斗日志组件说明

战斗日志组件负责把战斗的逐回合明细、掉落和结算放到网页里展示。消息正文只保留短摘要和 Markdown 改名链接，避免群消息被长日志刷屏。

## 命令

```text
战斗日志
战斗日志 开启 / 战斗日志 关闭
```

## 规则

- 玩家侧统一叫 `战斗日志`，不再使用其它名称。
- `战斗日志 开启` 表示消息里的战斗日志链接默认进入详细视图，逐次出手默认展开。
- `战斗日志 关闭` 表示消息里的战斗日志链接默认进入摘要视图，逐次出手默认折叠。
- 消息链接使用 Markdown 改名，例如 `[探险战斗日志〔123〕](完整地址)`，不展示裸 URL。
- 网页路径统一为 `/xiuxian/zhandou-rizhi/...`。
- 探险战斗日志读取 `exploration_records.record_id` 和 `result.events`。
- 切磋/决斗战斗日志读取 `duel_records.record_id` 和 `result`。
- 抢劫战斗日志读取 `robbery_records.record_id` 和 `result`。
- 首领战斗日志读取 `boss_challenge_records.record_id`，页面中附带首领事件和玩家累计参与数据。
- 虫洞战斗日志读取 `wormhole_challenge_records.record_id`，页面中附带虫洞事件和玩家累计参与数据。
- 战斗日志页面展示总览、掉落与结算、战斗明细三段内容。
- 战斗日志明细按短期流水处理，到期清理不影响长期统计、称号或人物志。
