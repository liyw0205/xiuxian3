# 修仙界历史组件说明

修仙界历史组件只负责全服公开信息：风云榜、修仙早报、修仙界历史和人物志。

## 命令

```text
风云榜
修仙早报
修仙界历史
人物志 玩家名称 / 人物志 [CQ:at,qq=对方ID]
```

## 规则

- 风云榜按当前业务日统计 Boss 伤害、明面资产、跑商、探险、珍稀武器和关键行为。
- 修仙早报每天缓存一份，栏目包括天地气象、今日最猛、今日最富、今日最勤、今日欧气、坊间传闻、商会风向、城池风向和首领动向。
- 修仙界历史按业务日沉淀世界记录，当前展示最近 5 天；当天会按实时数据刷新，旧日保持历史文本。
- 人物志可用玩家名称或 CQ/at 查询，只展示名称、称号、等级、常驻地点、代表武器和公开事迹，不展示 client_id 或详细资产。
- 玩家个人日记在 `玩家` 组件中处理。
- 今日气运只通过 `签到` 生成或查看。
- 世界天气和今日灵潮按业务日稳定生成，早报负责公开展示，实际加成由公共服务统一汇总。
- 武器传奇在 `武器` 组件中处理。
- 称号由公共服务按当前数据自动分配并自动佩戴，不提供手动查看或佩戴命令。
- 称号是玩家消息头和人物志里的近况窗口，来源覆盖当前资产、近期探险、跑商、武器状态、回收、虫洞、首领、对战和铭刻等行为倾向。
- 城池风向只展示 3-5 行摘要，例如最高建设城池、药路最紧城池、民生最盛城池、藏宝图动向和战备待牵引势力。
- 世界物资通知只放需要玩家处理或会错过的事项：藏宝图竞拍将结束、藏宝图可领取、宗主收到藏宝图、战备虫洞开启、虫洞奖励待领。
- 药路短缺、建设进度、民生高低、神秘蓄能未满不进通知栏，只进入地点文本或修仙早报。

## 扩展

- 需要按钮时统一使用 `修仙/markdown_utils.py`，例如 `button("修仙信息")`。
- 调整榜单和早报文案：改 `XiuxianHistoryService._build_newspaper()` 和对应排行函数。
- 调整大事记内容：改 `XiuxianHistoryService._build_chronicle_day()`。
- 调整人物志展示：改 `XiuxianHistoryService.profile()`。
- 大事记存放在 `schema_meta`，不需要新增表。
- 调整气运：改 `common.py` 的 `FORTUNE_POOL`。
- 调整称号规则：改 `common.py` 的 `CoreService.refresh_titles_conn()`。
- 调整城池风向：读取 `city_world_states`、`world_material_records`、`treasure_maps`、`war_prep_states` 和 `wormholes` 的懒结算结果。
