# 缘契组件说明

缘契是一次性角色剧情体验组件，使用祈愿产出的 `缘契凭证` 开启。它只提供体验入口，不保存网页进度，不上传结局，也不结算奖励。

## 命令

```text
开启缘契
开启缘契 书页醒来
查看所有缘契
缘契列表
```

发送 `开启缘契` 会消耗 `缘契凭证` x1，并生成一个 24 小时有效的随机剧本 `缘契开启码`。发送 `开启缘契 剧本标题` 可定向生成某个剧本的开启码，例如 `开启缘契 书页醒来`。开启码绑定当前修仙角色显示名。

发送 `查看所有缘契` 可查看当前可用剧本；列表展示名使用剧本 `game.title`，也就是玩家看到的 `书页醒来` 这类标题。

## 网页入口

```text
GET  /xiuxian/yuanqi
POST /xiuxian/yuanqi/start
```

网页需要填写角色名和缘契开启码。角色名必须和生成开启码时的角色显示名完全一致。校验成功后，服务端立即把开启码标记为已使用；若开启码绑定了指定剧本则返回指定剧本，否则随机返回一个启用剧本。刷新、关闭或网络中断都不会恢复这个开启码。

## 剧本

剧本放在：

```text
修仙/缘契/stories/剧本目录/story-data.json
```

当前所有存在的剧本都视为启用。无参数 `开启缘契` 进入时全随机抽取；带标题时会按剧本标题或目录 key 定向。第一版内置 `柳墟夜话`，并按世界皮肤的创作边界补充 `斗罗大陆`、`完美世界`、`百变小樱` 三套主题体验剧本；世界皮肤包只作为题材和命名参考，剧情文件仍统一维护在缘契组件内。

## 前端约束

运行页位于 `static/yuanqi`，文游核心文件复制在 `static/yuanqi/core`。本项目只通过 `yuanqi-driver.js` 接入后端，不依赖外部文游项目。

剧本文本可以使用 `{{name}}` 显示当前角色名。前端只替换标题、欢迎语、场景章节、时间、正文、对话、选项和通知消息等展示字段，不替换剧本 ID、场景 ID、跳转目标、状态路径、资源路径或条件字段。

## 音频

缘契剧情页可以使用 `bgm.play` 切换背景音乐。当前项目只放行 `/static/yuanqi/audio` 下的静态音频，剧本中的 `game.assetBaseUrl` 应指向这个目录，`audio.tracks.*.src` 只填写文件名，不填写外链、绝对路径或上级目录。

音频文件放在：

```text
static/yuanqi/audio
```

音源、作者和许可证记录在 `static/yuanqi/audio/SOURCES.md`。第一版音频均按 CC0 素材裁剪压缩，方便上线和后续替换。

乌鸦素材放在：

```text
static/yuanqi/audio/crow
```

这部分素材只做项目内通用精选库，前端提供“署名”按钮查看声明，项目内引用时保留“音乐由 乌鸦Producer 提供”。

## 边界

- 缘契凭证库存仍在 `wish_user_vouchers`。
- 开启码状态保存在 `yuanqi_codes`。
- 缘契不进入世界皮肤包，不影响玩家位置、探险、休息、战斗和收益曲线。
- 缘契没有排行榜、收藏册、成就、结局上报或奖励结算。
