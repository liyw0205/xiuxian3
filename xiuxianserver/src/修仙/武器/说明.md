# 武器组件说明

武器负责攻击力、自动技能、升级和附魔。

## 命令

```text
武器
切换武器 武器ID
升级武器 武器ID
附魔武器 武器ID 技能书名
铭刻武器 武器ID 新名字
铭刻附魔 武器ID 附魔序号 新名字
导航 铸剑阁
回收武器
回收武器 武器ID
```

## 规则

- 玩家没有武器时，会自动获得一把青岚短剑。
- 武器是主要攻击来源，人物自身只提供低基础攻击。
- 武器自带一个技能，战斗中按间隔自动释放。
- 武器类型会影响自动技能触发频率：轻武器更快，重武器更慢。
- 武器掉落时会随机品质和等级上限。
- 附魔栏由武器等级上限和当前等级共同决定。
- 武器列表会显示 `附魔:已用/已解锁`。
- 技能书不限制武器类型，附魔后永久占用附魔栏，不能撤销。
- 武器和已附魔技能书可以用铭刻之羽改显示名，只改显示，不改属性。
- 技能书各走一个专门方向，不做万金油加成。
- 附魔效果已参与打怪和玩家对战。
- 备用武器可在铸剑阁稳定回收。
- 单把回收价有上限，当天回收越多，后续估价越低。
- 已装备武器和最后一把武器不能回收。

## 回收估价

```text
原始估价 = (攻击 × 60 + 等级上限 × 100 + 当前等级 × 250 + 附魔数量 × 5000) × 品质系数 × 地点倍率
单把上限 = 12000 + 玩家等级 × 1800
最终回收价 = min(原始估价, 单把上限) × 今日回收倍率
```

今日回收倍率会随当天武器回收收入降低，最低保留 25%。这样武器回收可以兜底变现，但不会成为无限产源石入口。

## 附魔栏

武器 0 级自带技能不占附魔栏。
技能书不检查武器类型；同一本技能书可以附到任意武器上。
武器类型只影响最终触发间隔，匕首、飞刃更快，戟、斧、盘等更慢。
技能书只做一个清晰方向：命中、穿透、吸血、减伤、连击、频率或爆发，不做一书多用。

附魔栏只决定可以额外挂多少本技能书：

```text
max_level >= 20 且 level >= 20：解锁第 1 个附魔栏
max_level >= 40 且 level >= 40：解锁第 2 个附魔栏
max_level >= 60 且 level >= 60：解锁第 3 个附魔栏
max_level >= 80 且 level >= 80：解锁第 4 个附魔栏
max_level >= 100 且 level >= 100：解锁第 5 个附魔栏
```

示例：

```text
#1 青岚短剑[凡品] 已装备 等级:20/45 攻击:38 技能:风刃斩 附魔:0/1
#1 青云剑（青岚短剑）[凡品] 已装备 等级:20/45 攻击:38 技能:风刃斩 附魔:1/1（1.青云破（风刃书））
```

含义：

- 已用附魔栏：0
- 已解锁附魔栏：1
- 可以使用 1 本技能书附魔。
- 附魔后不可撤销，也不会返还技能书。
- 铭刻附魔时用武器列表里的附魔序号定位，例如：`铭刻附魔 武器#1 1 青云破`。

## 扩展

- 新增武器模板：改 `WEAPON_DEFS`。
- 新增武器技能：改 `WEAPON_SKILLS`。
- 新增附魔效果：改 `WEAPON_ENCHANTS`，并在 `combat_core.py` 里确认效果被读取。
- 调整武器类型频率：改 `WEAPON_TYPE_INTERVAL_FACTORS`。
- 调整升级费用：改 `weapon_upgrade_cost`。
- 调整回收金融曲线：改 `WEAPON_RECYCLE_*` 常量和 `weapon_recycle_price_rate`。
