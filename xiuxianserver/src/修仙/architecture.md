# 修仙模块使用和扩展教程

当前 `src/修仙` 是一个轻量玩法模块，二级包按玩法拆开：

- `玩家/`：创建用户、修仙信息、签到、新手礼包、休息。
- `源库/`：源库获取、查看、结息、升级、存入和取出源石。
- `天材地宝/`：查询修仙物品定义，统一说明背包物品和纳戒物品。
- `纳戒/`：保存不占负重的物品，主要是恢复类、宝石和技能书。
- `二手市场/`：玩家之间交易，每个玩家同一时间只能上架一包商品。
- `商场/`：系统跑商，买卖各地点特产。
- `铭刻/`：消耗铭刻之羽，为固定装备、武器和武器附魔改显示名。
- `异界虫洞/`：全服共享动态 Boss 命令入口。
- `首领/`：按节气和传统节日出现的岁时情劫，产出带文案的铭刻之羽。

## 命名规则

- 中文只用于业务模块目录名，例如 `玩家`、`源库`、`纳戒`、`商场`。
- Python 文件名、类名、函数名、变量名统一用英文。
- 玩家可见的命令和回复文本可以用中文。
- 只保留当前正式入口，目录结构错了就直接改成正确结构。

## 当前目录

- `__init__.py`：修仙根入口，暴露 HTTP `router`，注册数据库启动和关闭事件。
- `url.py`：HTTP 路由入口，目前是占位接口。
- `sql.py`：SQLite 数据层，只做建表和基础读写。
- `constants.py`：玩家、源库、商场的数值配置。
- `rules.py`：玩家、源库、商场的计算公式。
- `common.py`：玩家建档、源石、状态、日志、纳戒物品等共享能力。
- `item_effects.py`：可使用物品效果，背包和纳戒共用。
- `weapon_core.py`：武器实例、初始武器、掉落武器等公共能力。
- `combat_core.py`：打怪和玩家对战的公共结算。
- `wormhole_service.py`：异界虫洞生成、挑战、排行和奖励结算，供商场和异界虫洞命令入口共用。
- `玩家/`、`源库/`、`天材地宝/`、`纳戒/`、`商场/`、`铭刻/`、`首领/`：玩法模块包。

## 消息传播

1. 客户端连接 `/ws/bot/{client_id}`。
2. `client_id` 就是玩家唯一 id。
3. 客户端发送标准消息：

```json
{
  "code": 202,
  "type": "text",
  "message": "修仙信息",
  "request_id": "本次请求唯一 id"
}
```

4. ws 驱动拆出命令，命中对应 `@WsMessageHandler.handler(...)`。
5. 命令函数只接收自己需要的参数，当前修仙模块通常只用 `client_id` 和 `message`。
6. `message` 是触发片段后面的业务参数，例如 `存入源石 100` 触发后是 `100`。
7. 业务返回字符串后，命令函数通过 `send_reply` 发给当前 `client_id`。
8. `send_reply` 会给文本回复加上 `【玩家名】`，方便群聊里分辨是谁的结果。

没有命中的消息会立即返回 `code=404`，避免客户端一直等待。

## 本体和行商化身

- 本体负责探险、休息、首领、虫洞、切磋和赌约决斗。
- 行商化身负责跑商、导航、商场买卖、特殊出售和二手市场。
- 探险中可以跑商，因为行商化身在外界行动。
- 探险中不能挑战首领或虫洞，因为这类战斗必须本体亲自出手。

## 库存约定

背包保存占负重物品：

```text
client_id
item_id
quantity
```

背包物品占负重，适合跑商特产、材料和怪物战利品。

纳戒保存不占负重物品：

```text
client_id
equipment_item_id
quantity
```

纳戒物品不占负重，适合恢复类、宝石和技能书。

所有可直接使用并会增加经验、源石、血气、精神或洗髓体质的物品，都进入纳戒。

## 新增玩法

新增玩法时，新建一个中文二级包：

```text
灵宠/
  __init__.py
  service.py
```

`__init__.py` 放 ws 命令触发器，`service.py` 放业务逻辑。
共享能力只放根目录公共模块，比如 `common.py`、`weapon_core.py`、`combat_core.py`。
二级玩法包之间不要互相导入，需要共用能力时先下沉到根目录公共模块。
