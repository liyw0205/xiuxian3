# 探险组件说明

探险负责 30 分钟预计算、打怪、掉落和最终领取。

## 命令

```text
地点
地点列表
探险
探险状态
结束探险
掉落记录
```

## 流程

```text
探险 -> 预计算战斗和掉落 -> 死亡/背包满/30分钟到点停止 -> 探险状态可查看 -> 30分钟后结束探险领取
```

## 规则

- 开始探险后不会主动推送消息。
- 只有探险地点可以开始探险，特殊收购地点不能探险。
- 地点推荐等级只用于提示风险，不限制进入。
- 怪物按当前地点的怪物等级段生成，高级地点不会拦你，但会更危险。
- `探险状态` 可以提前查看预计算状态。
- `结束探险` 必须等 30 分钟冷却到点后才能结算。
- 30 分钟内的多场战斗会连续继承上一场剩余血气和精神。
- 死亡只会停止后续预计算，不删除角色。
- 背包格子或负重装不下下一件物品库掉落时，本次预计算提前停止。
- 当前地点特产会作为探险额外掉落来源。
- 恢复类、宝石、技能书进入纳戒，不占背包负重。
- 武器掉落会先记录结果，结束探险时才真正发放。

## 扩展

- 新增探险地点：改 `exploration_locations` 的初始化数据。
- 新增怪物：改 `MONSTER_DEFS`。
- 新增地点特色掉落：优先在商场特产里补齐，探险会按当前位置读取。
- 调整节奏：改 `EXPLORE_MINUTES` 和 `ENCOUNTER_SECONDS`。
